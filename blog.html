Discuss the scope of var, let, and const

Var: Variables declared with var are function scoped. No matter in what block or position of the function it's declared, it's hoisted to the top of the function and can be accessed from anywhere of the function. So, variables declared with var can be used before they are actually declared in the code. In this case the value will be undefined which leads to unnecessary confusions in code. That's why let is used now instead of var to avoid code confusions and to give improved scope control

Let: Variables declared with let are block scoped. They are only accessible within the block they are declared inside a function or code So, variables declared with let can be used only after they are actually declared in the code, and only from inside of the block it is declared.Outside the block the let variables are not accessible, as they are not Hoisted to top like variables declared with var.

Const: Variables declared with Const are block scoped like let variables. They are also only accessible within the block they are declared inside a function or code The difference of const from let is that const variables cannot be re-assigned after value is assigned once. BuT let variables gives flexibility to re-assign values as many times as wanted.

Example:
function scope() {
for (i = 3; i > 0; i--) {
let num1 = 3;
var num2 = 4;
const num3 = 5;
}
console.log(num1); //num1 will not be accessible outside the for loop block
console.log(num2); //num2 will be accessible outside the for loop block as it is hoisted on top of the function
console.log(num3); //num3 will not be accessible outside the for loop block
}

use cases of null and undefined

Runtime errors can be avoided by first determining whether a variable or property is null or undefined before using it. Few use cases of both kinds are:

Use Cases of Null:
1. To indicate that a variable or property has no meaningful value that we can use later in any conditional blocks, we can set its value to null.

2. Any value of a variable or property may be cleared using null.

3. We can initialise a variable to null, and update it with the actual data once it becomes available in cases, where data retrieval or initialization is asynchronous.

Use Cases of Undefined:
1. Variables that are not intialized are automatically set to undefined. We can use this in conditional blocks later to find which property or variables don't have values assigned and use them accordigly

2. Undefined is returned when we try to access a property of an object, that does not exist. This can be used in conditional blocks to determine if a property is present in an object or not and work accordingly

3. When calling a function with multiple parameters, if we only want to pass values for some of them, we can set the remaining arguments as undefined.

4. When trying to verify that an element exists inside an array or not, undefined will be useful here. When an element doesn't exist in an array, if we try to access it, it'll return undefined which we can use to verify that the element in not inside the array.
